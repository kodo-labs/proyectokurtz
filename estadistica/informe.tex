\section*{Introducción}
\addcontentsline{toc}{section}{Introducción}

El presente trabajo integrador tiene como objetivo aplicar herramientas de estadística descriptiva e inferencial al análisis de datos del sistema \textbf{BookDesk}, un sistema web de gestión de reservas para espacios de coworking desarrollado en el marco de la asignatura Ingeniería de Software II.

BookDesk permite a los usuarios reservar salas de reuniones y escritorios individuales, reemplazando la coordinación manual por una plataforma centralizada que previene conflictos de horarios y brinda visibilidad completa a miembros y administradores.

Para este análisis, se identificó la \textbf{duración de las reservas} (en horas) como variable principal, dado que es un indicador clave para la gestión eficiente de los recursos del coworking. Adicionalmente, se estudió la relación entre la cantidad de usuarios activos diarios y el número de reservas generadas.

Los datos fueron simulados a partir de patrones realistas del sistema, con una muestra de 50 observaciones para el análisis descriptivo y 30 observaciones para el análisis de regresión. El procesamiento se realizó mediante Python (numpy, pandas, scipy, matplotlib) y se integró directamente en el sistema BookDesk como un módulo de análisis estadístico.

\section{Recolección y organización de datos}

\subsection{Variable identificada}

\begin{itemize}[nosep]
  \item \textbf{Variable:} Duración de las reservas (en horas).
  \item \textbf{Tipo:} Cuantitativa continua.
  \item \textbf{Población:} Todas las reservas realizadas en el sistema BookDesk.
  \item \textbf{Muestra:} 50 observaciones simuladas con distribuciones realistas.
\end{itemize}

La variable fue seleccionada por su relevancia directa en la planificación del coworking: conocer cuánto duran las reservas permite optimizar la disponibilidad de salas y escritorios.

\subsection{Obtención de la muestra}

Se generó una muestra de $n = 50$ observaciones simulando los patrones de uso típicos del sistema:
\begin{itemize}[nosep]
  \item Reuniones cortas en salas (media $\approx$ 2h).
  \item Jornadas parciales en escritorios (media $\approx$ 4h).
  \item Jornadas completas en escritorios (media $\approx$ 6.5h).
  \item Reservas express en salas (media $\approx$ 1.5h).
\end{itemize}

\subsection{Organización de datos}

Los datos fueron organizados mediante un histograma de frecuencias que muestra la distribución de duraciones, y un gráfico de torta que muestra la proporción de reservas por tipo de recurso (salas de reuniones vs. escritorios individuales).

El histograma revela que la mayoría de las reservas se concentra en duraciones de 1 a 3 horas, con un segundo grupo en el rango de 5 a 7 horas correspondiente a jornadas completas. La distribución por tipo de recurso es equilibrada: 25 reservas en salas de reuniones y 25 en escritorios.

\subsection{Análisis del comportamiento}

La duración promedio de las reservas es de \textbf{3,29 horas}, con una mediana de 2,88 horas. La media es mayor que la mediana, lo que indica una \textbf{distribución con asimetría positiva}: hay reservas de larga duración (jornadas completas en escritorios) que elevan el promedio.

El coeficiente de variación es \textbf{55,62\%}, lo que refleja una alta dispersión. Esto es esperable dado que el coworking se usa tanto para reuniones breves de 1-2 horas como para jornadas completas de 6-7 horas.

El 50\% central de las reservas (entre $Q_1$ y $Q_3$) tiene una duración de entre 1,75 y 4,71 horas, con un rango intercuartílico de 2,97 horas. Esto sugiere que el sistema debería ofrecer slots flexibles que cubran desde reuniones rápidas hasta jornadas de trabajo extensas.

\section{Regresión y correlación}
% ═══════════════════════════════════════════════════════════════════════

\subsection{Variables seleccionadas}

Se analizó la relación entre dos variables del sistema BookDesk:
\begin{itemize}[nosep]
  \item \textbf{Variable independiente ($X$):} Número de usuarios activos por día.
  \item \textbf{Variable dependiente ($Y$):} Número de reservas diarias.
\end{itemize}

Se recolectaron datos de 30 días de operación del sistema. El objetivo es determinar si la cantidad de usuarios conectados permite predecir la demanda de reservas.

\subsection{Diagrama de dispersión}

El diagrama de dispersión muestra una clara tendencia lineal positiva: a medida que aumentan los usuarios activos, aumentan también las reservas diarias. Los puntos se agrupan de forma consistente alrededor de una línea recta ascendente.

\subsection{Ecuación de regresión lineal}

Aplicando el método de mínimos cuadrados se obtuvo:

$$\hat{y} = 2{,}1028 + 0{,}6028 \cdot x$$

\textbf{Interpretación:} Por cada usuario activo adicional que se conecta al sistema en un día, se esperan aproximadamente \textbf{0,60 reservas más}. La ordenada al origen (2,10) representa las reservas base esperadas incluso con actividad mínima.

\subsection{Coeficiente de correlación}

\begin{itemize}[nosep]
  \item \textbf{Coeficiente de Pearson:} $r = 0{,}9325$
  \item \textbf{Coeficiente de determinación:} $r^2 = 0{,}8695$
\end{itemize}

El valor de $r = 0{,}93$ indica una \textbf{correlación fuerte y positiva} entre las variables. El coeficiente de determinación $r^2 = 0{,}87$ significa que el \textbf{87\% de la variabilidad} en las reservas diarias queda explicada por la cantidad de usuarios activos.

El modelo resulta \textbf{útil} para predecir la demanda de reservas y podría integrarse en el dashboard administrativo de BookDesk para anticipar picos de uso.

\section{Estadística inferencial}
% ═══════════════════════════════════════════════════════════════════════

\subsection{Intervalo de confianza}

\textbf{Pregunta:} ¿Cuál es la duración media real de las reservas en BookDesk?

Se construyó un intervalo de confianza al 95\% para la media poblacional $\mu$:

\begin{itemize}[nosep]
  \item $n = 50$, $\bar{x} = 3{,}2946$ h, $s = 1{,}8323$ h
  \item Distribución: $t$-Student con $gl = 49$ ($\sigma$ desconocido)
  \item Valor crítico: $t_{0{,}025;\,49} = 2{,}0096$
  \item Error estándar: $\frac{s}{\sqrt{n}} = 0{,}2591$
  \item Margen de error: $E = 0{,}5207$
\end{itemize}

$$IC_{95\%}(\mu) = (2{,}77\text{ ; }3{,}82) \text{ horas}$$

\textbf{Interpretación:} Con un 95\% de confianza, la duración media poblacional de las reservas en BookDesk se encuentra entre \textbf{2,77} y \textbf{3,82 horas}. Esto permite al administrador planificar la disponibilidad de recursos con certeza razonable.

\subsection{Prueba de hipótesis}

\textbf{Pregunta:} ¿La duración media de las reservas es significativamente diferente de 3 horas?

\textbf{Formulación:}
\begin{align*}
H_0&: \mu = 3 \text{ (la duración media es 3 horas)} \\
H_1&: \mu \neq 3 \text{ (la duración media es diferente de 3 horas)}
\end{align*}

\textbf{Prueba bilateral con $\alpha = 0{,}05$:}

$$t_{calc} = \frac{\bar{x} - \mu_0}{s / \sqrt{n}} = \frac{3{,}2946 - 3{,}0}{1{,}8323 / \sqrt{50}} = 1{,}1369$$

\begin{itemize}[nosep]
  \item Valor crítico: $t_{crit} = \pm 2{,}0096$
  \item $|t_{calc}| = 1{,}14 < t_{crit} = 2{,}01$
  \item $p$-valor $= 0{,}2611 > \alpha = 0{,}05$
\end{itemize}

\textbf{Decisión:} \textbf{No se rechaza $H_0$}.

\subsection{Interpretación de resultados}

No existe evidencia estadística suficiente para afirmar que la duración media de las reservas sea significativamente diferente de 3 horas. Esto confirma que los slots de tiempo configurados actualmente en BookDesk son coherentes con el uso real del coworking.

Combinando ambos resultados: el intervalo de confianza ubica la media entre 2,77h y 3,82h (que incluye al valor 3), y la prueba de hipótesis confirma que no hay diferencia significativa respecto a 3h. Los slots de tiempo actuales están bien calibrados.

\section{Integración con Ingeniería de Software II}
% ═══════════════════════════════════════════════════════════════════════

El análisis estadístico realizado aporta información valiosa para la mejora y evolución del sistema BookDesk:

\begin{enumerate}[nosep]
  \item \textbf{Configuración de slots de tiempo:} La estadística descriptiva muestra que las duraciones varían ampliamente ($CV = 55{,}6\%$). Se recomienda implementar opciones predeterminadas de 1h, 2h, 4h y jornada completa para agilizar el proceso de reserva.

  \item \textbf{Predicción de demanda:} El modelo de regresión ($r^2 = 0{,}87$) permite estimar la cantidad de reservas según los usuarios activos. Se podría integrar un módulo predictivo en el panel administrativo que alerte cuando se espere alta demanda (por ejemplo, más de 25 usuarios activos implican más de 17 reservas).

  \item \textbf{Validación de políticas:} La prueba de hipótesis confirma que la configuración actual de slots de 3 horas es adecuada. Esta validación estadística reemplaza decisiones arbitrarias con respaldo cuantitativo.

  \item \textbf{Monitoreo continuo:} Los intervalos de confianza establecen un rango esperado (2,77h – 3,82h). Si el promedio de duración sale de este rango en períodos futuros, podría indicar un cambio en los patrones de uso que requiere atención administrativa.

  \item \textbf{Sugerencias automáticas:} El sistema podría sugerir duraciones de reserva basándose en el tipo de recurso y los patrones históricos, mejorando la experiencia del usuario.
\end{enumerate}
